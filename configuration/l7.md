# L7 Firewall Configuration

The `l7` section configures application-layer firewall rules evaluated in priority order (first-match-wins).

## Reference

```yaml
l7:
  rules:
    - id: "rule-id"
      priority: 10
      action: deny             # allow or deny
      protocol: http           # http, tls, grpc, smtp, ftp, smb, ssh,
                               # redis, mysql, postgres, dns-tcp, imap, pop3
                               # (enterprise adds: mqtt, amqp, nats, cassandra)
      # Protocol-specific fields (all optional):
      path: "/admin"           # HTTP path (regex)
      method: "POST"           # HTTP method
      host: "*.example.com"    # HTTP host
      sni: "*.malware.com"     # TLS Server Name Indication
      service: "pkg.Service"   # gRPC service
      share: "C$"              # SMB share name
      description: "Rule description"
```

## Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | `string` | Yes | Unique identifier |
| `priority` | `integer` | Yes | Evaluation order (lowest = first) |
| `action` | `string` | Yes | `allow` or `deny` |
| `protocol` | `string` | Yes | `http`, `tls`, `grpc`, `smtp`, `ftp`, `smb`, `ssh`, `redis`, `mysql`, `postgres`, `dns-tcp`, `imap`, `pop3` (enterprise: `mqtt`, `amqp`, `nats`, `cassandra`) |
| `description` | `string` | No | Human-readable description |
| `src_country_codes` | `[string]` | No | Source country filter (ISO 3166-1 alpha-2 codes). Rule matches only if the source IP's country is in this list |
| `dst_country_codes` | `[string]` | No | Destination country filter (ISO 3166-1 alpha-2 codes). Rule matches only if the destination IP's country is in this list |
| `src_ip` | `string` | No | Source CIDR filter |
| `dst_ip` | `string` | No | Destination CIDR filter |
| `dst_port` | `integer` | No | Destination port filter |

### Protocol-Specific Fields

| Protocol | Available Fields |
|----------|-----------------|
| `http` | `path`, `method`, `host`, `content_type` |
| `tls` | `sni` |
| `grpc` | `service`, `grpc_method` |
| `smtp` | `sender`, `recipient`, `command` |
| `ftp` | `command`, `path` |
| `smb` | `share`, `command`, `smb_command`, `is_smb2` |
| `ssh` | `service` (substring matched against the software banner) |
| `redis` | `command` (verb, case-insensitive), `path` (first-argument substring) |
| `mysql` | `smb_command` (command byte, e.g. `3` for COM_QUERY), `path` (SQL substring) |
| `postgres` | `command` (first char used as message type, e.g. `"Q"`), `path` (SQL substring) |
| `dns-tcp` | `host` (QNAME substring) |
| `imap` | `command` (verb, case-insensitive) |
| `pop3` | `command` (verb, case-insensitive) |

> The generic rule fields (`command`, `path`, `host`, `service`, `method`, `smb_command`) are reused across protocols. See the
> [L7 firewall feature doc](../features/l7-firewall.md#matcher-field-mapping) for the full table mapping protocol → field semantics.

### Enterprise extension protocols

MQTT, AMQP 0-9-1, NATS, and Cassandra CQL are detected by the enterprise `L7ExtendedParser` extension. The OSS dispatcher exposes these labels in metrics and alerts when the enterprise crate is loaded; dedicated matcher fields (MQTT topic patterns, AMQP virtual host filters, NATS subject wildcards, CQL query substrings) ship in a follow-up release.

## Examples

### Web application protection

```yaml
l7:
  rules:
    - id: block-admin
      priority: 10
      action: deny
      protocol: http
      path: "/admin.*"
      description: "Block admin panel"
    - id: block-internal-api
      priority: 20
      action: deny
      protocol: http
      path: "/api/internal/.*"
      description: "Block internal APIs"
    - id: block-bad-sni
      priority: 30
      action: deny
      protocol: tls
      sni: "*.malware-domain.com"
    - id: block-admin-share
      priority: 40
      action: deny
      protocol: smb
      share: "C$"
      description: "Block admin share access"
```

### GeoIP-based filtering

```yaml
l7:
  rules:
    - id: block-sanctioned-http
      priority: 5
      action: deny
      protocol: http
      src_country_codes: [KP, IR, SY]
      description: "Block HTTP from sanctioned countries"
    - id: block-tls-high-risk-dst
      priority: 6
      action: deny
      protocol: tls
      dst_country_codes: [KP, SY, CU]
      description: "Block TLS to high-risk destinations"
```
